UI Default interface kit for TurboSign & defui

Interface kit for viewer-style apps

Reusable pieces for PDF viewers and dashboard-like tools: buttons, cards, alerts, forms, and layout primitives. Copy any section into your own page and adjust tokens in ui.css.

Live preview

Think of this page as the “kitchen sink” for your UI: every section is a pattern you can reuse in the TurboSign viewer shell.

Actions

Buttons & toolbar patterns

Use .ui-btn plus modifiers for variants.

Primary actions

Use solid buttons for the main action in a toolbar or dialog, and ghost buttons for secondary or low-emphasis actions.

Combine a single .ui-btn--primary with one or two neutral / ghost buttons.

Reserve .ui-btn--danger for destructive, hard-to-reverse actions.

Toolbar layout

Combine a primary CTA, lightweight icons, and tabs or filters for a compact viewer toolbar. This mirrors the feel of a desktop app top bar.

Keep the toolbar dense but legible, mirroring desktop tooling rather than a generic marketing site navbar.

Feedback

Alerts & status

Stack alerts at the top of the canvas or in a dedicated activity rail.

✓
All recipients signed
The document is complete. You can download the executed copy or send a receipt.
!
Signature failed to save
The connection dropped while uploading the signature. Ask the signer to retry, or save their signature locally first.
i
Read-only mode
You opened an archived copy. To make changes, duplicate this document and work in the new copy.
